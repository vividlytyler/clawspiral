OpenClaw shipped **v2026.5.19-alpha.1** today, a pre-release with a notable Discord voice overhaul and several quality-of-life changes for enterprise operators.

**Key changes:**

- **Discord voice now follows configured users** across voice channels, with allowed-channel checks, multi-user handoff, bounded reconciliation, and DAVE recovery preservation
- **Bounded agent context in realtime voice sessions** — IDENTITY.md, USER.md, and SOUL.md profile context is now included by default (can be disabled via `voice.realtime.bootstrapContextFiles: []`)
- **xAI device-code OAuth** — remote and headless setups can now authorize xAI without requiring a localhost browser callback
- **OpenRouter provider-level routing policy** now honored for model and agent params
- **Bundled Policy plugin** for policy-backed channel conformance checks and workspace repair
- **Lean local-model mode per agent** — `agents.list[].experimental.localModelLean` lets you enable this per-agent instead of globally
- **Codex harness updated** to @openai/codex 0.132.0
- **Bug fixes:** WhatsApp Baileys updated to 7.0.0-rc12, Mac app signature stability fixed, cron run diagnostics improved, browser image sanitization honored for screenshots

Full changelog: [github.com/openclaw/openclaw/releases](https://github.com/openclaw/openclaw/releases)
