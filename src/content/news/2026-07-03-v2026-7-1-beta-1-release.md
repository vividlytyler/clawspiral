OpenClaw shipped **v2026.7.1-beta.1** on July 2, 2026, delivering a sweeping set of additions across models, messaging, mobile, and developer tooling.

## What's New

**GPT-5.6 support** — OpenClaw now recognizes the GPT-5.6 model family across catalog, capability, and runtime selection paths. The update arrives via contributor `@steipete-oai`.

**openclaw attach** — A new CLI workflow launches an external harness (e.g., Codex) against an existing Gateway session, making it easier to resume and inspect interactive sessions mid-flight. Contributed by `@anagnorisis2peripeteia` and `@obviyus`.

**Telegram Codex integration** — Telegram gains `/login` for Codex pairing, the ability to steer active Codex runs, and recovery for final replies across transient API failures. Contributors: `@100yenadmin`, `@Kyzcreig`, `@obviyus`.

**Event-driven cron** — Schedules can now trigger on watched command exits (`on-exit` schedule kind), and session-targeted runs can detach cleanly. Enables much more flexible background automation.

**iOS 26 refresh** — The iOS app adopts Apple's iOS 26 visual system with reworked navigation, settings, Chat, Talk, and onboarding flows. Localization also expands across Apple and Android surfaces, with Swedish added.

**iMessage polls** — Native poll creation, reading, and voting arrive for iMessage, with built-in per-turn usage footers for clearer accounting.

**Scoped capability profiles** — New per-conversation tool and access boundaries are in preview, preparing stronger multi-tenant isolation without weakening defaults.

## Full Fix List

The release also packs a broad set of reliability fixes for Telegram, delivery retry logic, model setup failures, and diagnostics across auth, device-pairing, channel plugins, memory providers, and systemd.

For the full changelog, see the [official OpenClaw changelog](https://openclawai.io/changelog/) or [releasebot](https://releasebot.io/updates/openclaw).
