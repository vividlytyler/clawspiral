OpenClaw's `2026.4.27` release landed today with a set of meaningful infrastructure improvements focused on GPU workloads, mobile device presence, and network security.

**GPU passthrough for sandboxed agents** — Docker sandbox containers can now opt in to `sandbox.docker.gpus` passthrough, enabling local GPU workloads to run inside sandboxed agents when the host Docker runtime supports `--gpus`. This unlocks heavier ML tasks in isolated contexts without breaking the sandbox boundary.

**Durable node presence for iOS and Android** — Background mobile nodes can now mark themselves recently alive without being treated as connected. Previously, backgrounding on mobile would cause nodes to drop from presence tracking; now they retain last-seen metadata after disconnects, making health checks and wake events more reliable.

**Outbound proxy routing** — Operators can enable `proxy.enabled` with `proxy.proxyUrl` / `OPENCLAW_PROXY_URL` for explicit forward-proxy routing. The implementation includes strict `http://` validation, loopback-only Gateway bypass, and cleanup of proxy env/dispatcher state on exit. Useful for enterprise environments with strict egress policies.

**Codex Computer Use** — Desktop agents can now run Codex Computer Use mode with `/codex computer-use status/install`, marketplace discovery, optional auto-install, and fail-closed MCP server checks before turns start.

**Other highlights:** non-image attachments via `chat.send`, refreshed provider dependencies (AWS SDK, Anthropic, Feishu), and Matrix streaming tool-progress live preview edits.

Release notes cover fixes across Active Memory, WhatsApp Web keepalive timeouts, Discord message reads, and security patches for npm execpath injection and device-pairing scope validation.
