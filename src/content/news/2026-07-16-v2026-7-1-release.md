OpenClaw v2026.7.1 shipped on July 13, 2026, marking one of the most substantial releases in the project's history — driven by 3,063 contributions from 532 external contributors.

## Control UI Overhaul

The browser-based Control UI received its most significant rewrite since launch. Sessions can now be pinned, grouped, renamed, forked, archived, and marked read. Multiple sessions display in resizable side-by-side panes that persist across reloads, and a live Tasks page lets operators inspect and cancel background work without leaving the UI. A redesigned composer keeps attachments, model selection, voice, and send/stop state usable across mobile and desktop screen sizes.

The Usage page got a comparable overhaul: recent estimated costs and daily values are easier to compare, provider-reported plans and billing details are surfaced, and seven-, thirty-, and ninety-day charts now show zero-activity days rather than dropping them.

## Native Mobile and Desktop Apps Upgraded

The official iOS, Android, and macOS apps received major updates alongside the web UI. Mobile pairing now works directly from Control UI — administrators can generate a setup QR code or copyable code from the Nodes page and use it to connect the iOS or Android app without manual gateway configuration.

## New Model Support: GPT-5.6, Tencent Hy3, Meta Muse Spark 1.1

The release adds first-class routing for several notable models:

- **GPT-5.6** — OpenAI's latest flagship, integrated with Codex harness defaults
- **Tencent Hy3** — Tencent's high-context reasoning model
- **Meta Muse Spark 1.1** — Meta's agent-oriented model, now routable through OpenClaw's model layer

## Stronger Codex and Coding-Agent Workflows

OpenClaw v2026.7.1 deepens integration with Codex and connected coding agents. The release notes reference "stronger Codex and connected coding-agent workflows" as a headline feature, building on earlier work that routed OpenAI model turns through Codex's native app-server harness.

## Messaging Platform Updates

Telegram, Slack, Discord, and Apple Messages each received substantial updates. The specific changes per platform span new message formatting, improved thread handling, and better approval and exec guardrail flows, though the full per-channel changelog is distributed across the GitHub PR references in the release notes.

## Underlying Improvements

Gateway crash loops, session routing, workspace terminal stability, remote browser control, scheduled work, and authentication failures all received targeted fixes. Path-routed Gateways now maintain the correct destination and credentials across restarts, and authentication failures show more actionable next steps rather than leaving pages stuck.

## Availability

OpenClaw v2026.7.1 is available now on GitHub, with Linux deb and AppImage packages now published as part of the stable release pipeline. Windows users can continue installing via winget immediately after Node.js is added.
