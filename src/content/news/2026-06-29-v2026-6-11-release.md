OpenClaw [v2026.6.11](https://github.com/openclaw/openclaw/releases) shipped with a cluster of channel control improvements that make multi-platform automation more flexible.

**Key highlights:**

- **Slack relay mode** — Native Slack integration now supports relay mode for more robust message handling across channels
- **Mattermost /oc_queue** — First-class Mattermost support with queue-based command handling
- **Per-DM model overrides** — Operators can now set different model preferences for individual direct message threads
- **openclaw agent --message-file** — File-driven agent input for workflows that pre-feed context
- **RAFT CLI wake bridge** — A new CLI mechanism for waking agents via the RAFT protocol

Community contributors @sjf-oa, @amknight, @xydigit-zt, @thomaszta, and @gandalf-at-lerian were acknowledged in the release notes.

The release follows a period of intense development for OpenClaw, which has seen rapid enterprise adoption alongside continued work on security hardening and skill ecosystem trust — including a recent NVIDIA SkillSpector collaboration for ClawHub skill verification.
