A new OpenClaw pre-release landed today with changes across onboarding, models, sessions, and cron.

**Key changes:**

- **Onboard/wizard:** Security disclaimer restyled with a single yellow warning banner, section headings, and bulleted checklists. Loading spinner added during initial model catalog load to prevent a blank screen. "API key" placeholder added to provider API key prompts.

- **Agents/prompts:** Default system prompt and OpenAI GPT-5 overlay strengthened with clearer completion bias, live-state checks, weak-result recovery, and verification-before-final guidance.

- **Models/costs:** Support for tiered model pricing from cached catalogs. Bundled Moonshot Kimi K2.6/K2.5 cost estimates added for token-usage reports.

- **Sessions/Maintenance:** Built-in entry cap and age prune now enforced by default. Oversized stores pruned at load time to prevent OOM from accumulated cron/executor session backlogs.

- **Cron:** Runtime execution state split into `jobs-state.json` so `jobs.json` stays stable for git-tracked job definitions.

- **Moonshot/Kimi:** Default bundled setup, web search, and media-understanding surfaces now use `kimi-k2.6` by default, with `kimi-k2.5` available for compatibility.

- **Agents/compaction:** Opt-in start and completion notices now sent during context compaction.

The full changelog is on GitHub: <https://github.com/openclaw/openclaw/releases>
