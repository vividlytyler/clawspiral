OpenClaw [v2026.5.12-beta.6](https://github.com/openclaw/openclaw/releases/tag/v2026.5.12-beta.6) has been released, addressing an iMessage-specific issue discovered and patched by community contributor @homer-byte.

## What Was Fixed

When sending native images via iMessage, OpenClaw was generating visible placeholder text for media-only sends. The fix removes this visible placeholder while preserving the internal echo key — the mechanism that prevents the agent from generating duplicate self-replies when it sees its own outgoing media.

## Scope

This is a targeted bug fix affecting the iMessage channel plugin. It does not appear to be a security-related release. Users of the iMessage integration who experienced unwanted placeholder text when sending images should upgrade to this pre-release to resolve the issue.

Check the [GitHub releases page](https://github.com/openclaw/openclaw/releases) for the full changelog and installation instructions.
