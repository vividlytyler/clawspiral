OpenClaw 2026.4.8 landed early April 8 with a sweep of bug fixes and new features.

**Channel setup fixes:** A packaging regression in recent releases caused installed npm builds to fail importing missing `dist/extensions/*/src/*` files during gateway startup across Telegram and 10 other bundled channels — BlueBubbles, Feishu, Google Chat, IRC, Matrix, Mattermost, Microsoft Teams, Nextcloud Talk, Slack, and Zalo. All now load through packaged top-level sidecars.

**New `openclaw infer ... hub` CLI:** A first-class hub subcommand for provider-backed inference workflows across model, media, web, and embedding tasks, contributed by @Takhoffman.

**Memory-wiki stack restored:** The bundled memory-wiki plugin is back — plugin, CLI, sync/query/apply tooling, memory-host integration, structured claim/evidence fields, compiled digest retrieval, claim-health linting, contradiction clustering, staleness dashboards, and freshness-weighted search.

**Media generation auto-fallback:** Provider switching for image, music, and video now falls back automatically across auth-backed providers, preserves intent during provider switches, remaps size/aspect/resolution/duration hints to the closest supported option, and surfaces provider capabilities.

**Session compaction checkpoints:** Operators can now inspect and recover pre-compaction session state via Sessions UI branch/restore actions.

**Webhook ingress plugin:** Bundled plugin so external automation can create and drive bound TaskFlows through per-route shared-secret endpoints.

**Slack proxy improvements:** Socket Mode WebSocket connections now honor ambient HTTP(S) proxy settings including NO_PROXY exclusions.
